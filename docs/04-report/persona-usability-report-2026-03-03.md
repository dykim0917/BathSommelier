## 3) 핵심 발견사항

### 강점

- 고위험군 안전 정책(임신/고혈압/숙취/당뇨) 적용이 일관적이다.
- 욕조/샤워/부분입욕 환경 오버라이드가 명확해 현실 제약 대응이 좋다.
- 입력이 비어 있거나 과도해도 추천이 안정적으로 수렴한다(회귀 테스트 통과).

### 개선 필요

- `왜 이 추천인가` 설명이 레시피 진입 전/후에 더 직접적으로 보여야 한다.
- 경고는 존재하지만 대안 행동(예: 수분 섭취, 온도 재확인, 중단 기준)의 즉시성/구조화가 부족하다.
- PRD 기준 미구현 intent(`cold_relief`, `menstrual_relief`, `stress_relief`, `mood_lift`)가 홈 커버리지를 제한한다.

## 4) 우선순위 액션

1. **P0**: 레시피 상단에 “추천 근거 2줄 + 안전 근거 1줄” 고정 노출
2. **P0**: 안전 경고를 단순 텍스트에서 “즉시 행동 체크리스트” 형태로 전환
3. **P1**: 미구현 Care intent 4종 우선 구현(홈 카드-엔진-서브프로토콜 일치)
4. **P1**: 샤워/부분입욕 전용 라벨 강화(환경 적합성 인지 개선)
5. **P2**: 초보 사용자용 기본 추천 안내 문구 추가(랜덤 인상 제거)

## 5) 결론

- 현재 버전은 **안전성과 추천 안정성은 높은 편**이다.
- 반면 **설명 가능성(Explainability)과 커버리지(미구현 인텐트)**가 체감 만족도를 제한한다.
- 위 P0/P1 항목을 반영하면, 신규 사용자 시작률과 고위험군 완주율 개선 가능성이 크다.
