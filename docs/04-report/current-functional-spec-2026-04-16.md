# Bath Sommelier 현재 기능명세서

- 작성일: 2026-04-16
- 기준 버전: 현재 코드베이스 기준 구현 명세
- 문서 목적: 외부 검증, 내부 QA, 기능 범위 확인
- 기준 원칙: 본 문서는 기획안이 아니라 현재 앱 구현을 역으로 읽어 정리한 "as-is" 기능명세서입니다.

## 1. 제품 개요

Bath Sommelier는 사용자의 목욕 환경과 건강 상태를 바탕으로 맞춤형 입욕 루틴을 추천하는 모바일 앱입니다.  
앱은 크게 `온보딩`, `홈`, `케어`, `트립`, `제품`, `프로필/기록`, `루틴 결과`, `타이머`, `완료` 흐름으로 구성됩니다.

핵심 제공 가치:

1. 사용 가능한 환경에 맞는 루틴 추천
2. 건강 상태를 반영한 안전 중심 추천
3. 케어형 루틴과 분위기형 트립 루틴의 이원화
4. 추천 시작부터 완료, 피드백, 기록 축적까지의 루틴 사이클 제공

## 2. 진입 및 라우팅

### 2.1 앱 최초 진입

- 저장된 프로필이 없거나 `onboardingComplete !== true`이면 `/onboarding`으로 이동합니다.
- 완료된 프로필이 있으면 `/(tabs)` 홈 탭으로 바로 진입합니다.

### 2.2 주요 라우트

- `/onboarding/welcome`: 온보딩 소개 화면
- `/onboarding`: 목욕 환경 선택
- `/onboarding/health`: 건강 상태 선택
- `/onboarding/greeting`: 온보딩 완료 화면
- `/(tabs)/index`: 홈
- `/(tabs)/care`: 케어 루틴 목록
- `/(tabs)/trip`: 트립 루틴 목록
- `/(tabs)/product`: 제품 추천 목록
- `/(tabs)/my`: 기록/설정 통합 화면
- `/result/recipe/[id]`: 루틴 준비/레시피 화면
- `/result/timer/[id]`: 루틴 진행 타이머 화면
- `/result/completion/[id]`: 완료 및 피드백 화면

## 3. 온보딩

### 3.1 웰컴 화면

- 브랜드 심볼과 앱 소개 문구를 노출합니다.
- 주요 가치 요약:
  - 환경 선택
  - 건강 상태 체크
  - 맞춤 루틴 추천
- CTA:
  - `시작하기`: 온보딩 진입
  - `이미 사용 중이면 바로 들어가기`: 탭 화면으로 우회 진입

### 3.2 목욕 환경 선택

- 사용자가 현재 가능한 환경을 선택합니다.
- 선택 가능한 값:
  - `욕조`
  - `족욕 (대야)` -> 내부적으로 `footbath` 성격이나 앱 UI에서는 `partial_bath` 계열로 운영
  - `샤워`
- 환경별 전용 아이콘 및 컬러 톤이 적용됩니다.
- 선택값은 다음 단계로 전달됩니다.

### 3.3 건강 상태 선택

- 복수 선택이 가능합니다.
- 선택 가능한 값:
  - `고혈압/심장`
  - `임신 중`
  - `당뇨`
  - `민감성 피부`
  - `해당 없음`
- `해당 없음`은 단독 선택 규칙을 가집니다.
- 다른 항목을 모두 해제하면 자동으로 `해당 없음`이 선택됩니다.
- 선택 완료 시 사용자 프로필이 저장됩니다.

### 3.4 온보딩 완료

- 환영 메시지와 함께 홈으로 진입하는 CTA를 제공합니다.
- 완료 시 `/(tabs)`로 이동합니다.

## 4. 홈 탭

홈은 오늘의 추천 진입 허브 역할을 합니다.

### 4.1 헤더

- 배지, 그리팅, 서브카피를 노출합니다.
- 일러스트 슬롯이 존재하며 에셋 교체 가능 구조입니다.

### 4.2 환경 전환

- 홈에서 즉시 환경을 전환할 수 있습니다.
- 선택 가능한 값:
  - `욕조`
  - `부분입욕`
  - `샤워`
- 선택값은 추천 카드 노출과 추천 생성에 즉시 반영됩니다.
- 최근 선택 환경은 별도 저장되어 다음 진입 시 복원됩니다.

### 4.3 주간 기록 카드

- 최근 완료 기록을 바탕으로 주간 루틴 달성 상태를 표시합니다.
- 포함 요소:
  - 주간 달성 수치
  - 진행 바
  - 오늘 완료 여부
  - 요일별 상태 표시
- 기록 보기 CTA를 제공합니다.

### 4.4 케어 루틴 섹션

- 상단 Hero 카드 1개와 하단 리스트 카드 3개 구조입니다.
- 기본적으로 현재 홈 케어 정렬 기준의 상위 카드가 Hero가 됩니다.
- `late_night` 시간대에는 `sleep_ready`가 허용 환경이면 Hero 우선 승격됩니다.
- Hero 카드 탭 시 `서브프로토콜 선택 모달`이 열립니다.
- 리스트 카드 탭 시에도 동일하게 케어 추천 생성 흐름으로 진입합니다.

### 4.5 트립 루틴 섹션

- 가로 스크롤형 카드 구조입니다.
- 홈에서는 `lite/deep 선택 모달` 없이 자동 서브프로토콜을 선택합니다.
- 트립 탭과 동일한 자동 선택 규칙을 사용합니다.
- 추천 생성 후 결과 화면으로 바로 이동합니다.

### 4.6 안전 안내

- 건강 상태에 따라 고정형 안전 고지 문구가 노출될 수 있습니다.
- 모든 추천은 레시피 화면에서 공통 프리플라이트 게이트를 먼저 노출합니다.
- 위험 조건이 포함된 추천은 공통 게이트 안에 추가 안전 항목이 붙습니다.

## 5. 케어 탭

케어 탭은 증상/상태 기반 회복 루틴 목록입니다.

### 5.1 환경 전환

- `욕조`, `부분입욕`, `샤워` 중 선택 가능합니다.

### 5.2 케어 인텐트 카드

현재 구현된 대표 케어 인텐트:

1. 근육 이완
2. 수면 준비
3. 숙취 완화
4. 붓기 완화
5. 감기 기운 완화
6. 생리통 완화
7. 스트레스 완화
8. 기분 전환

각 카드는 다음 정보를 가집니다.

- 인텐트 제목
- 환경별 서브카피
- 허용 환경 여부
- 안전 fallback 여부
- 카드 위치 기반 랭킹

### 5.3 서브프로토콜 선택

케어 루틴은 인텐트별 세부 선택지를 제공합니다.

예시:

- 근육 이완: 하체 / 어깨·목 / 전신
- 수면 준비: 생각이 많음 / 예민함
- 숙취 완화: 두통·민감 / 몸이 무거움

서브프로토콜 선택 시 변경되는 값:

- 진행 시간 증감
- 행동 블록 추가
- 조명 문구 보정
- 특정 환경 권장 힌트 추가

### 5.4 추천 생성

- 케어 추천은 `healthConditions + intent + environment`를 기반으로 생성됩니다.
- 결과는 히스토리에 저장되고 세션 로그도 함께 기록됩니다.

## 6. 트립 탭

트립 탭은 여행/무드 기반 몰입형 루틴 목록입니다.

### 6.1 환경 전환

- 현재 구현상 `욕조`, `샤워`만 선택 가능합니다.
- 부분입욕은 트립 전용 탭 UI에서 직접 선택하지 않습니다.

### 6.2 트립 인텐트 카드

기본 제공 테마 예시:

1. Kyoto Forest
2. Nordic Sauna
3. Rainy Camping
4. Snow Cabin

추가 트립 카드는 생성 카탈로그를 통해 확장될 수 있습니다.

### 6.3 자동 서브프로토콜

- 트립 루틴은 카드 탭 즉시 자동 옵션을 선택합니다.
- 별도 선택 모달 없이 결과를 생성합니다.
- 선택 결과는 추천 객체에 `subProtocolId`로 반영됩니다.

## 7. 제품 탭

제품 탭은 초보자가 바로 사용하기 쉬운 제품 카탈로그 중심입니다.

### 7.1 제품 목록

- 카탈로그 hydration 결과를 기반으로 제품을 표시합니다.
- 입문자 친화 제품만 우선 노출합니다.

### 7.2 카테고리 필터

- 전체
- 에센셜 오일
- 배스 솔트
- 배스 아이템
- 바디워시
- 허브

### 7.3 제품 상세 모달

- 제품 기본 설명
- 배지/아이콘
- 사용 맥락
- 구매 링크

### 7.4 외부 구매 링크

- 구매 URL이 없으면 안내 alert를 노출합니다.
- 링크 앱 실행이 불가하면 실패 alert를 노출합니다.
- 가능하면 외부 브라우저 또는 링크 앱으로 연결합니다.

## 8. 프로필 탭

프로필 탭은 `기록`과 `설정`의 2개 서브 탭으로 구성됩니다.

### 8.1 기록 탭

- 저장된 추천 이력을 2열 카드로 표시합니다.
- 필터:
  - 전체
  - 케어
  - 트립
- 각 기록 카드는 다음 정보를 포함합니다.
  - 추천명 또는 테마명
  - 모드
  - 온도
  - 목욕 타입
  - 날짜
  - 회상 메모 여부
- 카드 탭 시 해당 추천의 레시피 화면으로 이동합니다.
- 상단에 주간 streak 요약이 노출됩니다.
- 트립 메모리와 테마 선호도를 기반으로 인사이트가 계산됩니다.

### 8.2 설정 탭

- 현재 저장된 환경과 건강 상태를 수정할 수 있습니다.
- 항목 탭 시 즉시 저장됩니다.
- 앱 버전과 앱명 정보를 노출합니다.
- 재온보딩 reset UI는 코드상 존재하지만 현재 노출은 비활성화되어 있습니다.

## 9. 추천 결과 흐름

### 9.1 레시피 화면

- 추천 ID로 저장 이력을 조회해 상세 내용을 렌더링합니다.
- 제공 정보:
  - 추천명
  - 모드
  - 환경
  - 목욕 타입
  - 권장 온도
  - 권장 시간
  - 추천 이유
  - 준비 단계
  - 준비물/제품 매칭
  - 안전 안내
- 히스토리에서 재진입한 경우 `다시 시작` 확인 alert 후 타이머로 이동합니다.

### 9.2 제품 매칭

- 추천 성분과 환경 기반으로 연결 가능한 제품을 제안합니다.
- 추천 제품 상세 모달과 제품 탭 하이라이트 이동을 지원합니다.

## 10. 타이머

### 10.1 시작 방식

- 레시피 화면에서 시작 시 타이머 모달로 진입합니다.
- 짧은 intro phase 이후 자동 시작됩니다.
- intro 중 탭하면 즉시 시작할 수 있습니다.

### 10.2 진행 중 기능

- 남은 시간/총 시간 표시
- 진행률 기반 시각 효과
- 물/스팀 애니메이션
- 음악 + 앰비언스 동시 재생
- 볼륨 조절
- 일시정지 / 재개
- 조작 UI 숨김/표시

### 10.3 세션 저장

- 시작 시 활성 세션을 저장합니다.
- 완료 시 실제 소요 시간을 기록합니다.

## 11. 완료 화면

### 11.1 완료 요약

- 시간대 기반 메시지 노출
- 이번 달 완료 횟수 표시

### 11.2 피드백

- `좋아요 / 아쉬워요` 피드백을 1회 입력할 수 있습니다.
- 입력 시 추천 기록과 세션 로그가 갱신됩니다.

### 11.3 메모리 저장

- 완료 시 트립 메모리 히스토리가 저장됩니다.
- 저장 내용:
  - 완료 시각
  - 실제 소요 시간
  - 환경
  - 추천 온도
  - feedback
  - 회상 문구
- 테마가 있는 추천은 선호도 weight가 함께 누적됩니다.

### 11.4 완료 후 이동

- `기록 보기`
- `홈으로`

## 12. 안전 규칙

현재 구현된 핵심 안전 규칙:

1. 고혈압/심장 질환: 최대 38°C 제한
2. 임신 중: 최대 38°C 제한 + 특정 오일 제외
3. 음주 후: 미온수 족욕만 허용
4. 당뇨: 수온 확인 경고
5. 민감성 피부: 자극 성분 제외 경고

안전 조건은 다음에 영향을 줍니다.

- 최대 권장 수온
- 금지 성분 제외
- 강제 목욕 타입
- 경고 문구 노출
- 안전 우선 fallback 전략

## 13. 데이터 저장

로컬 저장소 AsyncStorage 기준으로 다음 데이터를 저장합니다.

1. 사용자 프로필
2. 추천 이력
3. 최근 선택 환경
4. 활성 세션
5. 트립 메모리 이력
6. 테마 선호도 가중치
7. 세션 로그

보존 규칙:

- 추천 히스토리 최대 30건
- 트립 메모리 최대 40건

## 14. 현재 구현 기준 제한사항

1. 사용자 계정, 서버 동기화, 멀티 디바이스 동기화는 없습니다.
2. 설정 탭의 재온보딩 reset 기능은 코드상 준비되어 있으나 현재 노출되지 않습니다.
3. 트립 탭 환경 선택은 욕조/샤워 중심으로 제한됩니다.
4. 추천은 카드 중심 진입 구조이며 자유 입력형 증상 문진은 현재 없습니다.
5. 히스토리 탭 전용 라우트는 별도 화면이 아니라 프로필 탭으로 redirect됩니다.

## 15. 검증 시 권장 확인 포인트

1. 환경과 건강 상태 변경이 추천 결과에 실제 반영되는지
2. 안전 규칙이 온도/성분/타입 제한에 반영되는지
3. 홈, 케어, 트립 진입 방식이 서로 다르게 동작하는지
4. 결과 저장, 타이머 완료, 피드백, 기록 반영까지 데이터가 이어지는지
5. 히스토리와 메모리 기반 인사이트가 누적 동작하는지
