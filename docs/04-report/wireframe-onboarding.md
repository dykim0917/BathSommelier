# 온보딩 화면 와이어프레임 (현재 구현 기준)

기준:
- `app/onboarding/index.tsx` (환경 선택)
- `app/onboarding/health.tsx` (건강 상태 선택)
- `app/onboarding/greeting.tsx` (환영/시작)
- `app/index.tsx` (초기 라우팅)

## 1) 목적
- 사용자 기본 프로필(목욕 환경, 건강 상태) 수집
- 안전한 추천을 위한 최소 입력 확보
- 입력 완료 후 메인 탭으로 진입

## 2) 전체 플로우
1. 앱 첫 진입(`app/index.tsx`)
- `onboardingComplete` 프로필 없음: `/onboarding` 이동
- 프로필 있음: `/(tabs)` 이동

2. Step 1 환경 선택 (`/onboarding`)
- 욕조 / 족욕(대야) / 샤워부스 중 1개 선택
- `다음` 탭 → `/onboarding/health?environment=...`

3. Step 2 건강 상태 선택 (`/onboarding/health`)
- 다중 선택 태그
- `완료` 탭 시 프로필 저장 후 `/onboarding/greeting`

4. 환영 화면 (`/onboarding/greeting`)
- 환영 메시지 + `시작하기`
- `시작하기` 탭 → `/(tabs)` replace

## 3) 화면별 구조

### A. 환경 선택 (Step 01/02)
1. 헤더
- 타이틀: `나의 목욕 환경을 알려주세요`
- 서브카피: 환경 맞춤 추천 안내

2. 환경 카드 리스트 (단일 선택)
- 카드 3종: 욕조 / 족욕(대야) / 샤워부스
- 선택 상태: 배경/보더 강조 + 체크 아이콘

3. 하단
- 스텝 인디케이터: `01 / 02`
- CTA: `다음` (선택 전 비활성)

### B. 건강 상태 선택 (Step 02/02)
1. 상단
- 뒤로가기 버튼
- 타이틀: `건강 상태를 선택해주세요`
- 서브카피: 안전한 입욕법 안내

2. 조건 태그 영역 (다중 선택)
- `고혈압/심장`, `임신 중`, `당뇨`, `민감성 피부`, `해당 없음`
- 규칙:
  - `해당 없음` 선택 시 단독 선택
  - 다른 조건 선택 시 `해당 없음` 자동 해제
  - 마지막 선택 해제 시 `해당 없음` 자동 복원

3. 하단
- 스텝 인디케이터: `02 / 02`
- CTA: `완료` (선택 전 비활성)

### C. 환영/전환 화면
1. 배경
- 블루 계열 그라디언트 + 장식 요소

2. 중앙 콘텐츠
- `환영합니다!`
- 메인 카피: `당신만의 목욕 루틴을 시작해볼까요?`
- 서브카피: 맞춤 레시피 안내
- 이모지 비주얼(🛁)

3. 하단 CTA
- `시작하기` 버튼
- 탭 시 메인 탭으로 이동

## 4) 상태 규칙
- 환경 화면:
  - 선택값 없으면 `다음` 비활성
- 건강 화면:
  - 선택값 없으면 `완료` 비활성
  - 완료 시 저장 프로필:
    - `bathEnvironment` = step1 선택값
    - `healthConditions` = 선택 태그 배열
    - `onboardingComplete = true`
- 환영 화면:
  - `시작하기` 탭 시 onboarding stack 종료 후 탭 진입(`replace`)

## 5) 텍스트 와이어프레임
```text
[Step 01: 환경]
  나의 목욕 환경을 알려주세요
  환경에 맞는 최적의 레시피를 추천해드립니다

  [🛁 욕조            ]
  [🦶 족욕(대야)      ]
  [🚿 샤워부스        ]

  01 / 02
  [다음]

[Step 02: 건강 상태]
  [<] 건강 상태를 선택해주세요
  안전한 입욕법을 위해 해당 사항을 모두 선택해주세요

  [⚠️ 고혈압/심장] [🤰 임신 중]
  [🩸 당뇨]       [🌵 민감성 피부]
  [✅ 해당 없음]

  02 / 02
  [완료]

[Greeting]
  환영합니다!
  당신만의 목욕 루틴을 시작해볼까요?
  맞춤 레시피와 함께 편안한 입욕 시간을 만들어 드릴게요
  🛁
  [시작하기]
```
