# Open Tab Header Pattern

## 목적

홈을 포함한 주요 탭의 상단 헤더를 카드형 컨테이너에서 분리해, 더 열린 구조의 헤더로 정리한다.

이번 변경의 핵심은 `헤더는 열린 구조`, `카드는 컨텐츠 전용`이라는 규칙을 화면 전반에 분명하게 적용하는 것이다.

## 문제 정의

현재 탭 화면의 상단은 `heroCard` 또는 `headerBlock` 형태로 구성되어 있어, 실제 컨텐츠 카드와 같은 시각 문법으로 읽힌다.

이로 인해 다음 문제가 발생한다.

- 페이지 진입 직후 정보 위계가 흐려진다.
- 헤더와 컨텐츠가 같은 무게로 느껴진다.
- 첫 인상이 답답하고 닫혀 보인다.

## 사용자 의도

사용자가 탭 상단에서 가장 먼저 알아야 하는 것은 아래 두 가지다.

- 지금 어떤 페이지인지
- 이 페이지에서 무엇을 할 수 있는지

따라서 헤더는 카드보다 가볍고 빠르게 읽혀야 하며, 실제 데이터/행동 블록보다 앞에서 방향을 잡아주는 역할에 집중해야 한다.

## 선택한 방향

공통 오픈 헤더 패턴을 정의하고, 모든 주요 탭에 일괄 적용한다.

이 방향을 선택한 이유는 다음과 같다.

- 문제의 원인이 개별 화면이 아니라 탭 상단 패턴 전체에 있기 때문이다.
- 일부 화면만 수정하면 다시 일관성이 깨질 가능성이 높다.
- `헤더는 열린 구조 / 카드는 컨텐츠 전용`이라는 규칙을 시스템으로 고정하는 편이 더 안정적이다.

## 대안 검토

### 대안 1. 헤더 카드만 부분 제거

- 장점: 빠르고 안전하다.
- 단점: 탭 간 일관성이 약하고, 패턴 정리가 충분하지 않다.

### 대안 2. 공통 오픈 헤더 패턴 일괄 적용

- 장점: 위계와 톤을 동시에 정리할 수 있다.
- 장점: 이후 탭 확장 시에도 재사용하기 쉽다.
- 단점: 수정 파일 수가 늘고, 간격 조정까지 함께 필요하다.

### 대안 3. 홈만 별도 헤더로 차별화

- 장점: 홈 허브 성격을 살리기 쉽다.
- 단점: 규칙이 복잡해지고, 다시 예외가 생긴다.

## YAGNI 정리

### Must Have

- 홈, 케어, 트립, 제품, 마이 탭의 상단 카드형 헤더 제거
- 각 탭 상단을 공통 구조의 오픈 헤더로 통일
- 헤더 정보는 `페이지명 + 한 줄 설명` 중심으로 단순화
- 첫 번째 컨텐츠 카드와 헤더 사이 간격 재조정
- 카드 스타일은 실제 컨텐츠 블록에만 유지

### Nice to Have

- 홈 헤더에 보조 문구 또는 메타 정보 1줄 유지
- 공통 헤더 스타일을 작은 재사용 패턴으로 정리

### Won't Do

- 새 인터랙션 추가
- 카드 시스템 전체 리디자인
- 탭 하단 컨텐츠 구조 전면 개편
- 새 디자인 토큰 도입

## 요구사항

### Functional

- 모든 주요 탭은 카드형이 아닌 오픈 헤더를 가진다.
- 오픈 헤더는 페이지명과 기능 설명을 즉시 읽을 수 있어야 한다.
- 헤더 아래 첫 번째 컨텐츠는 카드로 시작하되, 헤더와 시각적으로 분리되어야 한다.
- 기존 탭 기능과 동작은 바뀌지 않아야 한다.

### Non-Functional

- 탭 간 헤더 구조와 톤은 일관되어야 한다.
- 시선 흐름은 `헤더 → 첫 컨텐츠 카드`로 자연스럽게 이어져야 한다.
- 상단 밀도는 줄이되, 차분하고 고급스러운 인상은 유지해야 한다.
- 불필요한 리디자인 없이 최소한의 구조 변경으로 해결한다.

## 성공 기준

- 사용자가 탭에 들어왔을 때 페이지 제목을 더 빨리 인지할 수 있다.
- 상단이 카드에 갇혀 보이지 않아 첫 인상이 덜 무겁다.
- 첫 번째 실제 컨텐츠 카드가 더 명확하게 읽힌다.
- 탭들 사이에서 헤더 패턴이 일관되게 느껴진다.
- 기존 기능 동작은 바뀌지 않고 시각적 위계만 개선된다.

## 리스크와 대응

### 리스크 1. 헤더를 너무 비우면 페이지 성격이 약해질 수 있다

- 대응: `페이지명 + 한 줄 설명 + 필요 시 작은 eyebrow`까지만 유지한다.

### 리스크 2. 탭마다 높이와 간격이 달라 오히려 어수선해질 수 있다

- 대응: 공통 간격 규칙과 타이포 계층을 먼저 정한 뒤, 예외를 최소화한다.

### 리스크 3. 홈은 정보량이 많아 상단이 허전해질 수 있다

- 대응: 홈에도 같은 규칙을 적용하되, 보조 문구 1줄 정도는 허용한다.

### 리스크 4. 카드 제거 후 여백 밸런스가 깨질 수 있다

- 대응: 헤더 제거와 동시에 상단 패딩, 첫 섹션 시작 간격, 섹션 간 vertical rhythm을 함께 조정한다.

## 구조 원칙

- 헤더는 배경 위에 직접 놓이는 열린 레이아웃으로 구성한다.
- 카드는 실제 데이터, 선택, 추천, 기록 등 컨텐츠 영역에만 사용한다.
- 탭별 헤더는 같은 구조를 쓰되, 각 화면의 역할에 맞는 한 줄 설명만 다르게 둔다.
- 홈도 예외로 두지 않되, 허브 성격을 해치지 않는 수준의 보조 정보는 유지 가능하다.

## 구현 메모

- 가능하면 공통 헤더 스타일 또는 경량 재사용 패턴으로 정리한다.
- 기존 `heroCard`, `headerBlock` 등 카드형 헤더 래퍼는 제거하거나 역할을 축소한다.
- 첫 번째 컨텐츠 섹션이 실제 시작점처럼 읽히도록 간격과 정렬을 다시 맞춘다.
